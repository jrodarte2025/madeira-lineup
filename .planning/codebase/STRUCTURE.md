## Key File Locations

**Entry Points:**
- `src/main.jsx`: Initial React mount point; imports and renders MadeiraLineupPlanner component to DOM element with id="root"
- `public/index.html`: Not present in repo; auto-generated by Vite; contains single `<div id="root"></div>`

**Configuration:**
- `vite.config.js`: Minimal Vite configuration; only configures React plugin
- `eslint.config.js`: ESLint rules for code quality
- `package.json`: Dependencies (Firebase, React, React DOM), dev dependencies (Vite, ESLint, TypeScript)

**Core Logic:**
- `src/MadeiraLineupPlanner.jsx`: Single-file application containing:
  - Formation definitions (FORMATIONS constant)
  - Roster state management
  - Lineup state management
  - All UI components (PitchSVG, FieldPosition, PlayerChip, SaveLoadModal, RosterContent)
  - Event handlers (drag/drop, click, formation change, save/load/share/print)
  - Responsive layout logic
- `src/firebase.js`: Firestore initialization and two functions:
  - `loadPublishedLineup()`: Fetch published lineup from Firestore
  - `savePublishedLineup()`: Save lineup snapshot to Firestore

**Styling:**
- All styling inline via style objects; no separate CSS files
- Global styles injected as <style> tag in MadeiraLineupPlanner component (lines 755-771)
- Media queries for responsive design and print view

**Assets:**
- `public/madeira-fc-logo.png`: Madeira FC logo; referenced as `/madeira-fc-logo.png`
- `src/assets/`: Secondary images (vite.svg, react.svg, hero.png); currently unused in app

## Naming Conventions

**Files:**
- React components: PascalCase with .jsx extension (`MadeiraLineupPlanner.jsx`)
- Utility modules: camelCase with .js extension (`firebase.js`, `main.jsx`)
- Build/config files: kebab-case (`vite.config.js`, `eslint.config.js`)

**Directories:**
- Lowercase: `src/`, `public/`, `dist/`, `.planning/`, `node_modules/`
- Semantic naming: `codebase/` for docs, `assets/` for images

**JavaScript Identifiers:**
- Constants: UPPER_SNAKE_CASE (INITIAL_ROSTER, FORMATIONS, C for color palette)
- Functions: camelCase (encodeLineup, decodeLineup, buildShareUrl, shareLineup, loadPublishedLineup, etc.)
- Variables: camelCase (roster, activeHalf, modalOpen, etc.)
- React components: PascalCase (MadeiraLineupPlanner, PitchSVG, FieldPosition, PlayerChip, etc.)

**DOM/Styling:**
- CSS classes: `.screen-view`, `.print-view` (kebab-case, prefixed with function)
- Inline style objects: camelCase keys (backgroundColor, borderRadius, etc.)
- Color variables: Object `C` with semantic names (C.navy, C.orange, C.white, C.orangeGlow, etc.)
