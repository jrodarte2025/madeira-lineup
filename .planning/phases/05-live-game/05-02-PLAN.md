3. Build `src/games/LiveGameScreen.jsx` with timer and state machine:
   - Uses `useParams()` to get gameId from URL
   - On mount: call `loadGame(gameId)` from Firestore to get initial game data
   - Define `loadStored` and `saveStored` helpers (same pattern as MadeiraLineupPlanner)

   **Game State Shape:**
   - `gameStatus`: "setup" | "1st-half" | "halftime" | "2nd-half" | "completed"
   - `score`: { home: number, away: number }
   - `opponent`: string
   - `fieldPositions`: array from lineup snapshot (positions with players)
   - `benchPlayers`: array of players not on field
   - `events`: array of game events (stats, subs)
   - `halfIntervals`: array of { startAt: timestamp, endAt: timestamp|null }
   - `playerIntervals`: { [playerId]: [{ inAt: timestamp, outAt: timestamp|null }] }
   - `halfStartTs`: timestamp when current half started (for timer)

   **Timer (drift-proof, GAME-04):**
   - Use `useRef` for `startTsRef` (wall-clock timestamp) and `rafRef` (animation frame ID)
   - `startHalf()`: sets `startTsRef.current = Date.now()`, saves to localStorage, begins rAF loop
   - `tick()`: computes `elapsed = Math.floor((Date.now() - startTsRef.current) / 1000)`, sets `displaySeconds` state (only update state if value changed to avoid re-render spam), calls `requestAnimationFrame(tick)`
   - Display logic: if elapsed < 1500 (25 min), show countdown "MM:SS" where MM:SS = 1500 - elapsed. If elapsed >= 1500, show stoppage "+MM:SS" where seconds = elapsed - 1500
   - On unmount: `cancelAnimationFrame(rafRef.current)`

   **State Machine Transitions:**
   - "setup" → "1st-half": Coach taps "Start Game" (button shown when status is setup). Sets halfStartTs, pushes { startAt: Date.now(), endAt: null } to halfIntervals, initializes playerIntervals for all field players with { inAt: Date.now(), outAt: null }. Calls startHalf(). Updates Firestore status.
   - "1st-half" → "halftime": Coach taps "End Half" (only visible during stoppage). Cancels rAF, closes current halfInterval (set endAt = Date.now()), closes all active playerIntervals (outAt = Date.now()). Updates Firestore status.
   - "halftime" → "2nd-half": Coach taps "Start 2nd Half". Sets new halfStartTs, pushes new { startAt, endAt: null } to halfIntervals, re-opens playerIntervals for all currently-on-field players. Calls startHalf(). Updates Firestore status.
   - "2nd-half" → "completed": Coach taps "End Game" (only visible during stoppage). Cancels rAF, closes intervals. Updates Firestore status. Clears localStorage game keys.

   **Pitch Display (basic — no subs or stats yet):**
   - Render PitchSVG with FieldPosition circles for each field position from the lineup snapshot
   - Bench strip below header: horizontal scrollable row of player chips (same pattern as lineup tab bench scrubber)
   - FieldPosition onClick and drag handlers will be wired in Plans 05-03 and 05-04 — for now, just display players
   - Do NOT implement drag-and-drop substitution or stat selection in this task

   **Layout (top to bottom):**
   - GameHeader (fixed top)
   - Bench strip (below header, scrollable horizontal chips)
   - Pitch with FieldPositions (fills remaining space)
   - Bottom area reserved for StatBar and EventsFeed (Plans 05-03, 05-04)
