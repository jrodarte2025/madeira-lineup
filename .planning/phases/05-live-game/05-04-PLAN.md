  <action>
1. Add `selectedPlayerId` state (string|null) to LiveGameScreen:
   - Set via onClick on FieldPosition — only when game is active ("1st-half" or "2nd-half") and player is on the field (not bench)
   - Tapping the same player deselects. Tapping a different player selects the new one.
   - Clear selection when game transitions state (halftime, completed)
   - Pass `isSelected={player?.id === selectedPlayerId}` prop to each FieldPosition

2. Determine selected player's position group:
   - When selectedPlayerId is set, find which field position they occupy, get the position label, call `getPositionGroup(label)` to get the group
   - Pass the group to StatBar as `positionGroup` prop

3. Handle stat recording (`onStatTap` callback):
   - Import `abbreviateName` from `src/shared/utils.js`: `import { abbreviateName, getPositionGroup, calcMinutes } from "../shared/utils";`
   - Create a stat event: `{ id: crypto.randomUUID(), type: "stat", playerId: selectedPlayerId, playerName: abbreviateName(player.name), stat: statKey, half: currentHalf, t: Date.now() }`
   - Append to local events array (setState)
   - Call `appendGameEvent(gameId, event)` fire-and-forget (Firestore persistence)
   - Save events to localStorage immediately
   - Do NOT clear selection after recording a stat (coach often records multiple stats for the same player in quick succession)

4. Compute stat badge counts:
   - For each field player, count events where type === "stat" AND playerId matches AND half matches current half
   - Pass as `statCount` prop to each FieldPosition
   - Use useMemo keyed on [events.length, currentHalf] to avoid recomputation on every render

5. Wire EventsFeed:
   - Pass the full events array (EventsFeed slices to last 5 internally)
   - `onUndo` callback: remove event by id from local events array. This uses the `replaceGameEvents()` helper added in Plan 05-01 — do NOT try to use updateDoc/doc directly.
   - Import: `import { appendGameEvent, replaceGameEvents } from "../firebase";`
   - After undo: save updated events to localStorage

6. Layout integration:
   - EventsFeed positioned below the pitch, above StatBar
   - StatBar fixed at the bottom of the screen
   - Ensure the pitch area has enough bottom padding/margin to not be obscured by EventsFeed + StatBar
