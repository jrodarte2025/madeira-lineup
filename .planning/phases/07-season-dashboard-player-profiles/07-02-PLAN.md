  <name>Task 1: Implement StatsTab season dashboard with sortable table and accordion</name>
  <files>src/tabs/StatsTab.jsx, src/shared/summaryUtils.js</files>
  <action>
    Replace the StatsTab.jsx stub entirely with a full season dashboard component. If STAT_ORDER is not exported from summaryUtils.js, add the export there first.

    **State variables:**
    - `seasonData` (object|null) -- raw Firestore doc from loadSeasonStats
    - `seasons` (string[]) -- list of season IDs for dropdown
    - `currentSeason` (string) -- active season, default: getSeasonId(new Date().toISOString())
    - `sortKey` (string) -- "totalEvents" (default), "minutes", "gamesPlayed", or any stat key
    - `sortAsc` (boolean) -- false by default (highest first)
    - `expandedPlayerId` (string|null) -- which player row is expanded
    - `gamesByPlayer` ({}) -- lazy-loaded games cache: { [playerId]: gameDoc[] }
    - `loading` (boolean) -- true while fetching season data

    **On mount (useEffect):**
    1. Call loadSeasonStats(currentSeason) and listSeasons() in parallel
    2. Set seasonData, seasons, loading=false
    3. Ensure currentSeason always appears in dropdown even if not yet in Firestore

    **On season change:**
    1. Set loading=true, clear expandedPlayerId and gamesByPlayer
    2. Call loadSeasonStats(newSeason)
    3. Set seasonData, loading=false

    **Season selector dropdown:**
    - At top of component, styled consistently with app (navy bg, white text)
    - Shows formatted season labels: "Spring 2026", "Fall 2025" etc. (capitalize first letter + space + year)
    - Defaults to current season per getSeasonId

    **Dashboard table construction:**
    1. Derive activeCols: scan all players in seasonData.players for non-zero stat values; filter through STAT_ORDER for consistent ordering; exclude "minutes" and "gamesPlayed" from stat cols (they get their own columns)
    2. Build rows from INITIAL_ROSTER (all 13 players always shown):
       - For each player: look up `seasonData?.players?.[String(player.id)]` (String coercion critical -- Firestore keys are strings)
       - Compute totalEvents = sum of all stat values (excluding minutes and gamesPlayed)
       - Row shape: { player, stats, totalEvents }
    3. Sort rows by sortKey/sortAsc using the sort logic from RESEARCH.md Pattern 5
    4. Default sort: totalEvents descending

    **Table header row:**
    - Columns: Player | GP | Min | [dynamic stat cols using STAT_LABELS for display names]
    - Each header cell is tappable (onClick -> handleSort)
    - Sort indicator: show a small arrow (unicode up/down triangle) next to the active sort column
    - handleSort logic: if same column, toggle asc/desc; if new column, set to desc

    **Table data rows:**
    - Player column shows "#2 Alex Rodarte" format (jersey number prefix + full name per locked decision)
    - GP, Min, and stat cells show integer values (0 if no data)
    - Row is tappable -- onClick calls handleRowTap(player.id)
    - Subtle background highlight on the expanded row

    **Accordion expanded view (when expandedPlayerId matches):**
    - Renders below the player's row, spanning full table width
    - On first expand: lazy-load games via listGames(), filter to completed games in currentSeason using getSeasonId(game.date), cache in gamesByPlayer[playerId]
    - Show loading indicator while fetching
    - If player has 0 games in cache: show "No games played yet" message
    - For each game: build per-player row using buildSummaryRows(game), find the row matching the current player
    - Display: Opponent | Date (formatted short, e.g., "Mar 16") | Minutes | [stat values]
    - Each game row is tappable -- onClick navigates to /games/${game.id}/summary via useNavigate
    - Dynamic columns in expanded view: only stats with data for THAT player across their games
    - Collapse by tapping the same player row again

    **Empty states:**
    - No season data at all (seasonData is null AND no seasons exist): "No season stats yet -- finalize a game to start tracking"
    - Season data exists but a player has 0 everything: show the player row with all zeros (they still appear)
    - Expanded player with no games: "No games played yet"

    **Styling:**
    - Follow inline style patterns from GameSummaryScreen.jsx (tableStyle, thStyle, tdStyle patterns)
    - Navy background (C.navy), white text (C.white), fontBase typography
    - Table has horizontal scroll on small screens (overflow-x: auto on wrapper div)
    - Sticky first column (Player name) on horizontal scroll if feasible, otherwise acceptable to scroll
    - Orange accent (C.orange) for sort indicators and expanded row highlight
    - Compact cell padding for mobile friendliness
    - Accordion sub-rows slightly indented or lighter background to distinguish from main rows
