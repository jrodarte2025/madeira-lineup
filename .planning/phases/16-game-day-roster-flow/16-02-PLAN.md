<objective>
Make saved lineups behave as templates: their stored `inactiveIds` array is ignored on load, and when the game's per-game `inactiveIds` (set on the Game-Day Roster screen in plan 16-01) includes a player who was assigned to a position in the saved template, that position renders as a visually distinct empty slot ‚Äî coach manually fills it. Inactive players are already excluded from the bench (done in 16-01).

Purpose: Saved lineups are reusable starting-11 + formation templates. The coach should not have to remember which inactives were baked into the template and re-edit them ‚Äî the per-game choice always wins. Empty slots make the missing positions obvious during the pre-kickoff walkthrough.

Output: A `resolveLineupForGame` helper, GamesTab modal changes that strip `inactiveIds` from saved-lineup payloads at load time, and a LiveGameScreen field-position renderer that visually distinguishes empty slots from filled positions, plus the setup-state tap-to-fill persistence that closes the manual-fill loop.
</objective>

     if (!savedLineup) return null;
     const { inactiveIds: _drop, ...rest } = savedLineup;  // drop inactiveIds
     return rest;  // formation, lineup, roster, name preserved

   export function computeEmptySlotIndices(lineupArray, inactiveIds) {
     const inactives = Array.isArray(inactiveIds) ? inactiveIds : [];
     if (inactives.length === 0 || !Array.isArray(lineupArray)) return [];
