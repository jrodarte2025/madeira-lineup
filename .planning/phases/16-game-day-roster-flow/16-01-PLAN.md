<objective>
Insert a Game-Day Roster step into BOTH Start Game flows (GameSetupModal "Start Game Now" and GameDetailModal "Start Game"), persist per-game inactive selections to the game document's `lineup.inactiveIds` field, and make the live game's bench respect those inactives so they never appear as available substitutes.

Purpose: Inactive shifts from a global flag to a per-game decision made between game creation and kickoff. This unblocks plan 16-02 (saved-lineup-as-template loader, which needs the per-game `inactiveIds` write path to exist first) and plans 16-03/16-04.

Output: A new `GameDayRosterScreen` component, a new `updateGameInactives` Firestore helper, modal step-machine wiring in both GamesTab modals, and an updated bench builder in `LiveGameScreen` that filters by `inactiveIds`.
</objective>
