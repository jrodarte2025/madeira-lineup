<objective>
Ship a minimum viable quarter-based game flow: 4 x 12-min periods, clock auto-pauses at each period boundary, coach uses the existing tap-to-sub for mid-period subs. Gated on GAME_STRUCTURE so Madeira's halves behavior is byte-identical. 8-pre-built-segment-lineups and automatic 6:00 mid-quarter swaps are deferred to a future v3.1 phase if the friend's coach wants them.
</objective>

<approach>
Generalize the period lifecycle behind a small helpers module ‚Äî no schema changes, no lineup-planner changes, no Firestore migration. Every UI string and label is looked up by gameStatus + GAME_STRUCTURE so neither path branches with ad-hoc literals.

Key insight: `halfIntervals` is structurally a list of `{startAt, endAt}` ‚Äî it already works with any number of intervals. For quarters, we just append 4 entries instead of 2. Minute calc (interval intersection) works identically.

No existing Madeira game data changes shape. Existing Madeira games in Firestore (gameStatus="1st-half"|"halftime"|"2nd-half"|"completed"; halfIntervals with 2 entries) still load, render, and finalize exactly as before.
</approach>
