<objective>
Build a custom touch drag-and-drop system and mobile chip strip so coaches can drag players on their phones. The HTML5 drag API does not work on touch screens, so this plan implements touch event handlers (touchstart/touchmove/touchend) with visual ghost feedback and drop target highlighting. The chip strip replaces the bench bar's display function on mobile by showing unassigned players as tappable/draggable chips above the pitch.

Purpose: Coaches use this app at practice on their phones -- drag-and-drop is essential, not optional. This is the foundation for all mobile interactions.
Output: Working touch drag between chip strip and field, and between field positions, on mobile. Desktop unchanged.
</objective>

  <name>Task 1: Implement useTouchDrag hook and integrate with chip strip + field positions</name>
  <files>src/MadeiraLineupPlanner.jsx</files>
  <action>
Create a `useTouchDrag` custom hook inside MadeiraLineupPlanner.jsx (above the main component) that manages touch-based drag-and-drop. The hook should:

1. **State:** Track `touchDragState` with `{isDragging, playerId, source, ghostX, ghostY, overTarget}` where source is "roster" (from chip strip), a number (field position index), or null.

2. **handleTouchStart(playerId, source, e):** Called on touchstart of draggable elements. Store the playerId and source. Set a small timeout (~150ms) before activating drag to distinguish from taps. On activation, prevent default to stop scroll, create ghost state.

3. **handleTouchMove(e):** On touchmove, if drag is active: update ghostX/ghostY from `e.touches[0].clientX/Y`. Use `document.elementFromPoint(clientX, clientY)` to detect which drop target the finger is over. Set `overTarget` to the detected target's data attribute (e.g., `data-drop-id="field-3"` or `data-drop-id="chipstrip"`). Call `e.preventDefault()` to prevent scrolling during active drag.

4. **handleTouchEnd(e):** On touchend, if drag was active: read the current `overTarget`. Execute the appropriate action:
   - If overTarget is a field position (e.g., "field-3"): If source is "roster", call `assignPlayer(playerId, targetIdx)`. If source is a field index, call `swapPositions(source, targetIdx)`.
   - If overTarget is "chipstrip": If source is a field index, call `removeFromPosition(source)`.
   - Clear all drag state.
   If drag was NOT active (touch was too short), treat as a tap — call the existing click handler for that element.

5. **Ghost element:** Render a fixed-position div that follows the finger during drag. Show the player's number in a 50px circle matching the FieldPosition style (navy gradient, orange border). Position at ghostX/ghostY with transform translate(-50%, -50%). Only render when isDragging is true. Use a React portal or just a fixed div at the end of the component.

**Integration — chip strip above the pitch (mobile only):**

Add a new section in the JSX, rendered only when `isMobile`, placed ABOVE the pitch container in the column flow (between header and pitch). Structure:

```
<div data-drop-id="chipstrip" style={...}>
  {availablePlayers.length === 0
    ? <div>"All players assigned"</div>
    : availablePlayers.map(p => (
        <div key={p.id}
          data-drag-id={p.id}
          data-drag-source="roster"
          onTouchStart={(e) => handleTouchStart(p.id, "roster", e)}
          onClick={() => handlePlayerClick(p.id)}
          style={chipStyle}
        >
          <span>{p.num}</span> <span>{p.name.split(" ")[0]}</span>
        </div>
      ))
  }
  {availablePlayers.length > 0 && !touchDragState.isDragging && !selectedPlayer &&
    <div style={hintStyle}>Drag to position</div>
  }
</div>
```

Chip style: 44px min-height (Apple HIG), pill shape (borderRadius 22px), horizontal scroll with flexWrap: "nowrap", overflowX: "auto", gap: 8px. Each chip shows `num + firstName`. Selected chip gets `background: C.orange`. Chip strip area has subtle background `rgba(0,0,0,0.15)` and 1px bottom border. When `availablePlayers.length === 0`, show "All players assigned" message but keep the strip height consistent (min-height: 52px).

**Integration — field positions:**

Add touch handlers to each FieldPosition. In the FieldPosition component, add `onTouchStart`, `onTouchMove`, `onTouchEnd` props. On the outer div of FieldPosition:
- `onTouchStart={has ? (e) => onTouchStart(e) : undefined}` — only occupied positions are draggable
- Add `data-drop-id={`field-${idx}`}` attribute for drop target detection

In the parent, pass touch handlers:
```
onTouchStart={(e) => handleTouchStart(player.id, idx, e)}
```

**Drop target highlighting:**

When `touchDragState.overTarget` matches a field position's `data-drop-id`, apply the same orange glow as the existing `shouldGlow` logic. For the chip strip, apply `background: rgba(232,100,32,0.15)` and `border: 2px dashed C.orange` when overTarget is "chipstrip".

**Remove compact mode on mobile:**

Change line ~1003 from `compact={isMobile}` to `compact={false}` so field circles are always 50px.

**Desktop preservation (PRES-01):**

All touch handlers are ONLY attached on mobile. The existing HTML5 `draggable`, `onDragStart`, `onDragEnd`, `onDragOver`, `onDrop` handlers remain untouched for desktop. The chip strip only renders when `isMobile`. The existing bench bar at the bottom remains for now (will be hidden on mobile in Plan 02).

**Important implementation notes:**
- Use `{ passive: false }` when adding touchmove listener if using addEventListener directly, to allow preventDefault
- The ghost element must have `pointerEvents: "none"` so elementFromPoint sees through it
- Set `touch-action: none` on draggable elements via inline style to prevent browser scroll during drag
- The 150ms delay before activating drag prevents accidental drags during taps — if touchend fires before 150ms, treat as tap instead
