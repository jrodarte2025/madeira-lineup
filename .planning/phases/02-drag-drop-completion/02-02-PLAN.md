**Current behavior of handlePositionClick (line 747):**

This handles: bench player selected + click empty position = assign. But when a bench player is selected and the target is occupied, `assignPlayer` overwrites — which is actually the swap behavior (displaced player returns to bench automatically). So DND-02 click path may already work. However, we need to also handle:

**Field-to-field click swap (new):** When a field player is selected (selectedPlayer is in currentLineup) and user clicks a different field position, swap them.

Rewrite `handlePositionClick`:
```javascript
const handlePositionClick = (posIndex) => {
  if (!selectedPlayer) {
    // No selection — select the field player at this position (if any)
    if (currentLineup[posIndex]) setSelectedPlayer(currentLineup[posIndex]);
    return;
  }

  // A player is selected — determine source
  const selectedPosIndex = currentLineup.indexOf(selectedPlayer);
  const isSelectedOnField = selectedPosIndex !== -1;

  if (isSelectedOnField) {
    // Selected player is on the field
    if (selectedPosIndex === posIndex) {
      // Clicked same position — deselect
      setSelectedPlayer(null);
    } else {
      // Clicked different position — swap the two field positions
      swapPositions(selectedPosIndex, posIndex);
      setSelectedPlayer(null);
    }
  } else {
    // Selected player is on the bench — assign to this position
    // assignPlayer handles: puts bench player here, displaces occupant to bench
    assignPlayer(selectedPlayer, posIndex);
    // selectedPlayer is cleared inside assignPlayer
  }
};
```

**Double-click to remove (DND-01 click path):**
Already wired at line 956: `onDoubleClick={() => player && removeFromPosition(idx)}`. Verify this still works after the handlePositionClick changes (click selects, double-click removes — these should not conflict because double-click fires after the second click, and the first click selects the player, but the double-click handler should still fire removeFromPosition).

Potential issue: on double-click, the first click triggers `handlePositionClick` which selects the player, then the second click triggers `handlePositionClick` again which would either deselect (if same position — our new logic handles this by deselecting) OR the double-click event fires `removeFromPosition`. The event order is: click -> click -> dblclick. So after the first click, the player is selected. After the second click (same position), `selectedPlayer` is the same player, `selectedPosIndex === posIndex`, so we deselect. Then `onDoubleClick` fires `removeFromPosition(idx)`. This works: double-click = deselect then remove. The position is cleared.

Actually there's a subtlety: the second click sees `selectedPlayer === currentLineup[posIndex]` and `selectedPosIndex === posIndex`, so it calls `setSelectedPlayer(null)`. Then `onDoubleClick` fires `removeFromPosition(idx)`. That's correct behavior.

**No changes needed for bench-to-occupied click swap** — `assignPlayer` already handles it by overwriting the position.
